# AI Resume Agent - 项目叙事

## 一句话定位

**解决AI PM求职者面临的"技术深度表达"与"数据可信度"双重挑战。**

---

## 问题背景

### 目标用户痛点
1. **表达不够专业** - 普通简历优化工具不懂 RAG、LangChain、QPS 等AI术语
2. **数据可信度存疑** - AI容易编造量化指标（幻觉问题）
3. **缺乏针对性** - 市面工具太通用，无法体现AI PM岗位的特殊要求

### 现有解决方案的不足

| 方案 | 问题 |
|------|------|
| ChatGPT通用优化 | 不懂AI术语，优化太泛 |
| 普通简历工具 | 无技术深度，无反幻觉 |
| 找人代写 | 贵，且不懂AI行业 |

---

## 解决方案

### 核心思路
**不是做一个"更强的AI"，而是做一个"懂AI PM领域的专项优化器"**

### 4-Agent协作架构

```
JD输入
  ↓
┌─────────────┐
│  Analyst    │ → 解码JD中的核心技能、隐性需求、关键词权重
└─────────────┘
  ↓
┌─────────────┐
│  Optimizer  │ → 硬核STAR法则重构，要求量化数据占位
└─────────────┘
  ↓
┌─────────────┐
│  Verifier   │ → 反幻觉校验，对比原文标记新增数据
└─────────────┘
  ↓
┌─────────────┐
│  Recruiter │ → 生成面试问题 + 多维评分报告
└─────────────┘
```

### 关键设计决策

1. **硬核STAR法则**
   - 禁用"负责""参与"等弱动词
   - 要求"主导""重构""构建"
   - 必须包含工具链（LangChain+ChromaDB等）

2. **反幻觉Verifier**
   - 提取原文所有数据声明
   - 优化后逐条对比，标记"疑似编造"
   - 用户确认后才进入最终输出

3. **量化数据占位符**
   - 当原文无数据时，生成精确占位符
   - 格式：`[需补充QPS提升数据]`、`[在此处填入具体数值]`
   - 禁止使用"显著提升"等模糊词汇

## 技术架构

### 技术栈
- **后端**: FastAPI + Python
- **Agent**: 4-Agent协作系统（Analyst/Optimizer/Verifier/Recruiter）
- **LLM**: MiniMax API（支持温度控制）
- **前端**: 原生HTML/CSS/JS
- **导出**: Word文档生成

### 评分维度

| 维度 | 说明 |
|------|------|
| 逻辑性 | STAR结构完整性（情境→任务→行动→结果） |
| 技术深度 | JD技能关键词匹配度 |
| 业务贡献 | 量化指标密度 |
| 排版美观 | 结构化程度 |

## 差异化亮点（面试可强调）

### 1. Multi-Agent协作 ≠ 简单Prompt工程
> "很多人觉得AI应用就是写prompt，但真正的难点在于Agent之间的状态传递、反馈循环、和职责边界设计。我的4-Agent系统中，Verifier需要理解原文语义才能做反幻觉校验，这不是prompt能做到的。"

### 2. Anti-Hallucination是真实产品问题
> "AI PM在工作中经常遇到模型编造数据的问题。我在简历agent中引入了Verifier机制，要求每一项新增的量化指标都必须有原文依据。这不是学术设想，而是我预判到的真实用户痛点。"

### 3. 领域知识沉淀
> "我专门为AI PM岗位设计了硬核STAR法则——要求在情境中交代技术痛点（如模型幻觉、QPS瓶颈），行动中包含具体工具链，结果必须量化。这需要对AI行业有足够理解才能设计出来。"

### 4. 迭代优化闭环
> "不是一次生成就结束，而是Recruiter评分后反馈给Optimizer，最多迭代3轮。这个反馈循环的设计灵感来自敏捷开发——快速验证，持续改进。"

## 用户价值闭环

```
用户输入原始简历 + JD
       ↓
系统分析JD关键词权重
       ↓
生成优化后简历 + 面试问题 + 评分报告
       ↓
Verifier标记所有量化声明 → 用户确认
       ↓
导出Word文档直接投递
```

## 面试常见问题应对

### Q: 这个项目最大的挑战是什么？
**参考回答：**
> "最大的挑战不是技术实现，而是设计Verifier的判断逻辑——如何让系统能识别'疑似编造'的数据？我参考了RLHF中的人类反馈机制，设计了一套基于原文比对的校验流程。"

### Q: 为什么不用LangChain而要自己写Agent逻辑？
**参考回答：**
> "LangChain是应用层框架，适合快速开发。但我的需求是定义Agent之间的协作协议和状态传递格式，这部分需要自己设计。LangChain可以later集成进来，但核心逻辑我选择自己实现以便更好地控制。"

### Q: 这个项目体现了你哪些产品能力？
**参考回答：**
> "第一，需求洞察——我观察到AI PM简历优化的特殊需求；第二，系统设计——多Agent如何分工协作；第三，反幻觉意识——这是我预判的AIGC合规风险；第四，用户价值闭环——从输入到导出到确认，每一步都有明确的用户价值。"

## 下一步优化方向

### Phase 2: 段落优化器（待开发）

**场景**：用户只有零散的经历要点，不知道怎么组织成简历格式

**新增入口**：首页 tab 切换
```
[ 完整简历优化 ] [ 段落优化 ]
```

**输入**：
- 目标 JD（同一份）
- 经历要点列表（如：3-5个bullet points）

**输出**：
- 扩展后的完整 STAR 段落（带标注）
- 可一键复制

**示例**：
```
输入要点：
- 做了数据分析和可视化
- 独立负责产品上线
- 获得了用户好评

输出：
[原文]独立负责产品上线[/原文]，结合[推理]数据分析和可视化结果[/推理]优化产品决策，
通过[补充]A/B测试验证假设[/补充]，最终产品上线后[需补充]请填写具体数据和效果[/需补充]。
```

**改动文件**：
- `frontend/index.html` - 新增 tab 切换和段落优化输入区域
- `api_server.py` - 新增 `/api/optimize-section` 接口
- `agents/optimizer.py` - 复用，新写一个处理要点扩展的 prompt

### Phase 3: 其他功能
- PDF/Word 简历直接解析
- ChromaDB 存储历史优化案例
- 用户反馈循环优化模板
- 面试模拟（语音交互）
